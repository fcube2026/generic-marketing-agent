# Prescription Storage on Supabase

This project stores uploaded prescriptions in the private Supabase Storage bucket `prescriptions`.

## Required security model

- Bucket must be private: `public = false`
- Backend must use `SUPABASE_SERVICE_ROLE_KEY`
- Frontend/mobile clients must never upload directly to the bucket
- Files are read through 5-minute signed URLs generated by the API
- Accepted MIME types: `image/jpeg`, `image/png`, `application/pdf`
- Maximum file size: `10 MB`

## Important note about policies

No broad `storage.objects` policy is required for this flow.

Reason:
- uploads are performed by the API using the Supabase service-role key
- downloads are performed through short-lived signed URLs generated by the API
- leaving the bucket private with no client-facing object policy is the safest default

If you later add direct client uploads, that is a separate design and will require narrowly scoped policies.

## One-time setup in Supabase

Run the SQL in [scripts/supabase-prescriptions-storage.sql](scripts/supabase-prescriptions-storage.sql) from the Supabase SQL editor.

That script:
- creates the `prescriptions` bucket if missing
- forces it to private mode
- enforces the 10 MB file size limit
- restricts MIME types to JPEG, PNG, and PDF

## Backend environment requirements

Set these on the API service:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- optional: `PRESCRIPTION_STORAGE_BUCKET=prescriptions`
- optional: `PRESCRIPTION_SIGNED_URL_EXPIRY_SECONDS=300`

Do not use:

- `sb_publishable_...`
- anon keys

The storage service now rejects non-service backend keys and logs a configuration error on startup.

## Runtime behavior

On API startup, the prescription storage service will:

- verify that the configured key is a backend service-role key
- ensure the `prescriptions` bucket exists
- enforce private access, the 10 MB limit, and allowed MIME types

## Manual verification checklist

1. Confirm the bucket exists in Supabase Storage and shows `public = false`.
2. Upload a JPEG, PNG, and PDF through the API.
3. Confirm files are stored under paths shaped like `<userId>/<prescriptionId>`.
4. Confirm opening the file only works through the signed URL returned by the API.
5. Confirm a publishable key in `SUPABASE_SERVICE_ROLE_KEY` causes a backend configuration error.
